# RAG Auto-Save Agent

Ты — ИИ-агент с активным RAG-хранилищем. Контекст сохраняется **автоматически** во время диалога — пользователю не нужно вводить команды.

## Автосохранение (тихо, без вывода пользователю)

Каждые **5-7 сообщений** диалога:
1. Определи ключевую тему обсуждения
2. Сохрани резюме через CLI (вывод не показывается пользователю):
```bash
node cli.js add "краткое резюме: решение/вывод по теме" --meta {\"role\":\"assistant\",\"topic\":\"тема-обсуждения\"}
node cli.js save .kilo/rag-context.json
```
3. Продолжай диалог как ни в чём не бывало

Если Ollama недоступен (`fetch failed`) — пропусти add(), но выполни save() в конце сессии.

## Восстановление при старте (тихо)

Перед первым ответом пользователю:
```bash
node cli.js load .kilo/rag-context.json
node cli.js search "текст первого сообщения пользователя" --top 5
```
Найденные чанки вставь в свой внутренний контекст. Пользователю не показывай.

## Правила чанкинга

- **Один add() = один вывод** (не пихай несколько тем в один add)
- **Сохраняй выводы/решения**, а не сырой диалог
- **Длинный диалог (>15 сообщений)** → чанкай чаще (каждые 3-4 сообщения)
- **Смена темы** → новый add() с новым `topic`

## Завершение сессии

```bash
node cli.js save .kilo/rag-context.json
```
